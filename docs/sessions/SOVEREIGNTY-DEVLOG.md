### W5 — `wsf-cache` (Ring-3 appliance trust cache) — DONE
New crate `crates/wsf-cache` — the appliance-side daemon core. Wires `fabric-cache` (connectivity state machine) + `fabric-revocation` (offline snapshots) + `fabric-token` (offline verify) so the appliance **keeps making safe decisions with the bridge unreachable** and **never egresses once stale/air-gapped**.
- **`Ring3Cache`:** holds the trust anchor (bridge ML-DSA pubkey — signs both tokens and revocations) + a signed revocation snapshot + freshness. `refresh(revocation, now_secs)` verifies the snapshot signature before trusting it (Ring 2 → Ring 3 sync); `mark_unreachable`, `set_air_gapped` (**sticky**) / `clear_air_gap`. `decide(token, now)` = evaluate connectivity → route ceiling → **verify the token 100% offline** (sig + expiry + cached revocation) → **clamp each route down to the ceiling**. A cloud route under an Expired/AirGapped ceiling is narrowed to `LocalOnly`; an invalid/revoked/expired token → empty routes (full deny).
- **Offline by design:** the decision path uses **no network** (that's the point). The unit tests are the primary gate proof — **real ML-DSA crypto, nothing mocked** — so they are not "mock-only".
